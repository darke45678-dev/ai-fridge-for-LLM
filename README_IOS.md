# 🚀 Kitchen AI - iOS 實體 App 安裝指南 (免 App Store)

本指南旨在幫助您將開發好的 **Kitchen AI** 專案，繞過 Apple App Store 的審核與限制，直接以「實體安裝檔」的方式載入您的 iPhone 進行使用。

## 🛠️ 第一階段：取得專案與自動化編譯 (GitHub Actions)

我們已經在 GitHub 倉庫中配置了 **GitHub Actions (雲端 Mac 編譯工廠)**。每當您推送程式碼，GitHub 會自動幫您打包 iOS 專案。

1. **前往 GitHub 倉庫**：訪問 [AI-fridge-IOS](https://github.com/darke45678-dev/AI-fridge-IOS)。
2. **尋找 Actions**：點擊頂部的 `Actions` 標籤。
3. **下載編譯產物**：
   - 點擊最近一次成功的 `Build iOS IPA (Standalone)` 任務。
   - 在頁面最下方 **"Artifacts"** 區域，下載 `KitchenAI-iOS-Build`。
   - *註：目前的腳本會產出一個 `App.xcworkspace` 專案包，供後續手動簽免使用。*

## 📲 第二階段：在 Windows 上「側載」安裝 (Sideloadly)

因為 iPhone 預設只允許安裝來自 App Store 的軟體，我們需要使用 **Sideloadly** 工具來幫您的 App 進行「個人簽名」。

1. **下載與安裝 Sideloadly**：
   - 前往 [Sideloadly 官網](https://sideloadly.io/) 下載並安裝。
2. **連接設備**：
   - 使用傳輸線將 iPhone 連接到 Windows 電腦。
   - 如果手機彈出「信任此電腦」，請點擊 **「信任」**。
3. **簽名與傳輸**：
   - 打開 Sideloadly，將您從 GitHub 下載的 `.ipa` 檔案（或編譯好的 App 成果）拖入。
   - 在 **"Apple ID"** 欄位輸入您的信箱。
   - 點擊 **"Start"**，輸入您的 Apple ID 密碼（這是由 Apple 官方驗證的，用來簽發 7 天的臨時開發者證書）。
4. **信任開發者 (手機端)**：
   - 安裝完成後，App 會出現在桌面，但點開會顯示「不受信任的開發者」。
   - 請前往 iPhone 的 **「設定」 > 「一般」 > 「裝置管理 / VPN 與裝置管理」**。
   - 點擊您的 Apple ID，然後點擊 **「信任 [您的 Apple ID]」**。

### 🔄 如何更新 App？
只要您在電腦端修改了 UI 或優化了模型，並執行以下指令推送到 iOS 倉庫：
```powershell

git commit -m "優化 AI 辨識邏輯"
git push ios_origin main
```
隨後 GitHub 會重新跑編譯流程，您只需重複「第一階段」的下載與「第二階段」的覆蓋安裝即可！
